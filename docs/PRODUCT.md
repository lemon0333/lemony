# lemony — 제품 정의 (PRODUCT)

## 문제 / 간극
노코드 AI 빌더(Lovable 등)는 두 개의 벽이 있다:
1. **시작의 벽** — 무엇을/어떻게 요청해야 할지 모른다(프롬프트 감각 필요).
2. **유지의 벽** — 프로젝트가 커지면 AI가 코드를 헷갈려 엉뚱한 수정을 하고, 비전공자는 고칠 수 없어 이탈.

lemony는 두 벽을 모두 낮춘다. 특히 (2)를 **Quarkify 신뢰성 엔진**으로 해결하는 것이 해자.

## 타겟 (페르소나)
- **주 타겟: 소상공인/1인 사업자** — 가게/포트폴리오/예약 페이지가 필요하지만 개발 못 함.
- 부 타겟: 블로거(기존 사이트 이어받기), 동아리/소규모 단체, 비개발 직군 기획자.

## Lovable 대비 차별점
| | Lovable류 | lemony |
|---|---|---|
| 자연어 | 영어 중심, 프롬프트 감각 필요 | 한국어 우선, 대화형 가이드 |
| 큰 프로젝트 수정 | 할루시네이션↑, 깨짐 | Quarkify 코드맵으로 정확/안전 |
| 기존 프로젝트 | 약함 | import→이해→이어작업 |

## 핵심 플로우
생성(자연어→사이트) → 인덱싱(Quarkify 맵) → 편집 루프(요청→타겟→수정→프리뷰) → 발행.

## MVP 범위
- 한국어 프롬프트로 단일 페이지(랜딩) 생성
- "버튼 더 크게/색 바꿔/문의 폼 추가" 수준의 자연어 편집
- 실시간 프리뷰 + 원클릭 발행(정적 호스팅)
- Quarkify 기반 편집 타겟팅(틀린 곳 고치지 않기)

## 북극성 지표
**수정 성공률(iteration success rate)** — 사용자의 자연어 요청 1회가 의도대로 반영된 비율. (할루시네이션 = 이 지표의 적)

## 리스크
- 생성 품질/디자인 일관성, 자연어 모호성, 비용(토큰), 발행/도메인 운영, Quarkify 인덱싱 지연.
